# alopex-cli 要求仕様（ドラフト）

## 目的とスコープ
- Alopex DB/Server/Cluster を CLI から操作するための統一インターフェースを定義する。
- ユースケース: ローカル開発/検証、運用監視、バッチ投入、トラブルシュート。

## 対象モード
1) **ローカルファイル/オブジェクトアクセス**
   - 対象: `file://`（ローカル）、`https://`、`s3://` 等で配置された `.alopex` ファイル群。
   - 方式: 埋め込み API（alopex-embedded）をクライアント内に同梱し、読み取り/書き込みを直接実行。
   - 要件:
     - ファイル/URL からセグメントをオープンし、KVS/カラムナ/ベクトル/SQL CRUD を実行可能。
     - 読み取り専用オプションと書き込み可能オプションを明示。
     - 遅延ロードや範囲指定ダウンロード（Range GET）による大容量最適化を行う。
     - `https://` へのアップロード（書き込み）は非対応。`s3://` でのアップロード/書き込みはクレデンシャルとポリシーに依存し、権限がない場合は明示的にエラーとする。

2) **Server モード**
   - 対象: 単一 Alopex Server への接続。
   - 方式: gRPC/HTTP API 経由で RPC を発行。認証・接続設定を CLI で管理。
   - 要件:
     - KVS/カラムナ/ベクトル/SQL CRUD API の操作を提供。
     - サーバー状態確認: バージョン、起動時間、メトリクス、ストレージ使用量、接続数。
     - 管理操作: 健全性チェック、設定読み出し、メンテナンス操作（例: compaction トリガ、インデックス再構築）。

3) **Cluster モード**
   - 対象: Chirps ベースのクラスタまたは将来の cluster API。
   - 方式: クラスタエンドポイントへ接続し、ルーティング/フェイルオーバーを CLI 側でハンドル。
   - 要件:
     - CRUD 操作はクラスタの適切なノードへルーティング（メタデータ取得 → ノード選択）。
     - 状態確認: メンバー一覧、ロール（leader/follower）、ヘルス、Raft/レプリケーションラグ、シャード配置。
     - 管理操作: ノード追加/削除、リバランス、スナップショット/バックアップ起動、クラスタ設定変更（認証/圧縮/レプリカ係数など）。

## 機能要求
- **共通 CRUD**
  - KVS: get/put/delete/scan、トランザクション（begin/commit/rollback）。
  - カラムナ: セグメント ingest、scan（projection/filter pushdown）、統計/インデックス確認。
  - ベクトル: upsert/delete、knn/search（metric: cosine/L2/inner）、フィルタ条件付き検索。
  - SQL: execute/query、バッチ実行、結果ストリーム（JSON/CSV/TSV 出力選択）。
- **接続管理**
  - プロファイル保存: `~/.alopex/config` に接続名ごとのエンドポイント、認証情報、デフォルト DB/テーブル。
  - 認証: トークン/Basic/mTLS。mTLS 用の cert/key/CA 設定と検証。
  - タイムアウト/リトライ/バックオフ設定。
- **入出力**
  - 出力フォーマット: table/JSON/quiet/CSV/TSV（シンプル運用＋CSV/TSV を優先、NDJSON 等は当面非対応）。エラーは構造化 JSON を選択可能。
  - バッチ入出力: ファイル指定（-f）、標準入力/標準出力パイプ対応。
  - ページング: 大量結果のページング、または `--limit/--offset`。
- **オブザーバビリティ**
  - `status`/`metrics` コマンドでメトリクス/統計表示。
  - `profile` オプションで API 呼び出し時間や I/O バイト数を表示。
  - ログレベル切替（quiet/info/debug/trace）。
- **安全性・整合性**
  - 破壊的操作には確認プロンプト（`--yes` でスキップ）。
  - トランザクションがサポートされないモードでは警告を表示。
  - バージョン互換性チェック（クライアント/サーバー/ファイルフォーマット）。

## コマンド体系（案）
- `alopex profile` … プロファイルの作成/一覧/削除。
- `alopex kv` … get/put/delete/scan/txn。
- `alopex columnar` … ingest/scan/stats/index/show-segment。
- `alopex vector` … upsert/delete/search/build-index/show-stats。
- `alopex sql` … query/exec、バッチ実行（-f）。
- `alopex status` … 接続先の状態確認（server/cluster）。
- `alopex admin` … compaction、バックアップ、ノード管理、設定変更。
- `alopex config` … 全体設定の確認/編集（出力形式、TLS、timeout 等）。

## 実装フェーズ案
- Phase 1（ローカル/オブジェクト）: `file://`/`s3://` 読み書き、KVS/カラムナ/ベクトル/SQL CRUD、出力フォーマット（table/JSON/quiet/CSV/TSV）。
- Phase 2（Server）: プロファイル管理、認証（token/Basic/mTLS）、サーバー状態確認（status/metrics）、compaction などの軽微管理コマンド。
- Phase 3（Cluster）: クラスタ接続・ルーティング、メンバー/ロール表示、レプリケーションラグ確認、ノード追加/削除やリバランスなどの管理操作。

## ロードマップ

### v0.x（現行バージョン）— バッチモードがデフォルト
- **デフォルト動作**: コマンド逐次実行（バッチモード）。
- **目的**: 基盤となる CLI 機能の安定化と API 互換性の確立。
- **含まれる機能**:
  - Phase 1〜3 の CRUD 操作、接続管理、認証。
  - 出力フォーマット（table/JSON/CSV/TSV）。
  - プロファイル管理、オブザーバビリティ機能。
- **TUI の位置づけ**: `--tui` / `-t` オプションで明示的に起動（プレビュー機能）。

### v1.0（次期メジャーバージョン）— TUI がデフォルト
- **デフォルト動作**: TUI モード（インタラクティブ）。
- **目的**: ユーザー体験の向上と探索的データ分析のサポート。
- **移行方針**:
  - バッチモードは `--batch` / `-b` オプションで引き続き利用可能。
  - 既存スクリプトは `ALOPEX_MODE=batch` 環境変数で互換性を維持。
  - TTY 検出により CI/CD 環境では自動的にバッチモードへフォールバック。
- **新機能**:
  - TUI 全機能（データ閲覧、検索・フィルタリング、インラインエディット）。
  - セッション管理（状態保存、複数タブ、履歴）。
  - テーマ・カスタマイズ機能。

### v1.x 以降（将来計画）
- **プラグインシステム**: カスタムコマンド/フォーマッタの追加。
- **リモートセッション共有**: 複数ユーザーによる同一セッションの閲覧。
- **AI アシスタント統合**: 自然言語によるクエリ生成支援。

### マイグレーションガイド（v0.x → v1.0）
| v0.x での使い方 | v1.0 での対応 |
|----------------|--------------|
| `alopex sql "SELECT ..."` | `alopex sql -b "SELECT ..."` または `ALOPEX_MODE=batch` 設定 |
| `alopex sql --tui` | `alopex sql`（デフォルト） |
| CI/CD スクリプト | 変更不要（TTY 検出で自動バッチモード） |
| パイプ入力 | 変更不要（自動バッチモード） |

## UX 要件
- サブコマンドと主要オプションは短縮エイリアスを用意（例: `-p` プロファイル、`-o` 出力）。
- ヘルプは詳細/簡易の2段階（`-h` / `--help --verbose`）。
- エラーは改善アクション付きで表示（例: 認証失敗時に必要なフラグを提示）。
- オートコンプリートスクリプト生成（bash/zsh/fish/pwsh）。

### 実行モード

alopex-cli は 2 つの実行モードを提供する。

#### TUI モード（デフォルト）
- **概要**: インタラクティブな TUI によるデータ操作モード。探索的なデータ分析や運用時の視認性に優れる。
- **起動**: 引数なしでコマンド実行、または明示的に `--tui` / `-t` を指定。
- **TTY 検出**: 標準入出力が TTY でない場合は自動的にバッチモードへフォールバック。
- 例: `alopex sql`、`alopex kv scan`、`alopex vector search`

#### バッチモード（CI/スクリプト向け）
- **概要**: 従来型の逐次実行モード。CI/CD パイプライン、シェルスクリプト、cron ジョブでの利用を想定。
- **起動**: `--batch` / `-b` オプションを指定、またはパイプ入力時に自動適用。
- **特徴**:
  - 結果を標準出力に直接出力（table/JSON/CSV/TSV 選択可）。
  - 対話的プロンプトなし（`--yes` 相当の動作）。
  - 終了コードによるエラーハンドリング（0: 成功、1: エラー、2: 警告）。
- 例: `alopex sql -b "SELECT * FROM users"`、`cat queries.sql | alopex sql -b`

#### 環境変数によるデフォルト設定
- `ALOPEX_MODE=batch`: デフォルトをバッチモードに変更。
- `ALOPEX_MODE=tui`: デフォルトを TUI モードに強制（TTY がない場合はエラー）。

### TUI 機能詳細
TUI によるインタラクティブなデータ操作モードを提供し、探索的なデータ分析や運用時の視認性を向上させる。

#### データ閲覧機能
- **テーブルビュー**: カラム幅の自動調整、水平/垂直スクロール、行番号表示。
- **詳細パネル**: 選択行の全フィールドをフォーマット済みで表示（JSON/YAML 切替可）。
- **カラム操作**: ソート（昇順/降順）、カラムの表示/非表示切替、カラム順序の入れ替え。
- **ページング**: 大量データの仮想スクロールまたはページ単位ナビゲーション。

#### 検索・フィルタリング機能
- **インクリメンタルサーチ**: `/` キーで検索モードに入り、入力に応じてリアルタイムでマッチ行をハイライト。
- **フィルタ式**: `:filter` コマンドで条件式を入力し、表示データを動的に絞り込み。
- **正規表現サポート**: 検索/フィルタで正規表現を利用可能。
- **検索履歴**: 過去の検索クエリを `↑`/`↓` で呼び出し可能。

#### データ管理機能
- **インラインエディット**: 選択行の値を直接編集（KVS put、ベクトル upsert 等に反映）。
- **バルク操作**: 複数行選択（Shift+矢印、Ctrl+A）後に一括削除/エクスポート。
- **クリップボード連携**: 選択範囲を JSON/CSV 形式でコピー（`y` キー）。
- **エクスポート**: 現在のビュー/フィルタ結果をファイルに保存（`:export` コマンド）。

#### キーボードナビゲーション
- **基本移動**: `j`/`k` または `↑`/`↓` で行移動、`h`/`l` または `←`/`→` でカラム移動。
- **ジャンプ**: `g` で先頭、`G` で末尾、`Ctrl+d`/`Ctrl+u` でページ移動。
- **ビュー切替**: `Tab` でパネル間移動、`1`-`9` でタブ/ビュー切替。
- **コマンドパレット**: `:` でコマンド入力モード、`?` でキーバインド一覧表示。
- **終了**: `q` または `Ctrl+c` で TUI を終了し、最後の状態を標準出力に表示（オプション）。

#### 視覚的フィードバック
- **シンタックスハイライト**: JSON/SQL の構文ハイライト表示。
- **ステータスバー**: 接続先、レコード数、フィルタ状態、処理中インジケータを常時表示。
- **通知エリア**: 操作結果（成功/エラー）を一時的にトースト表示。
- **テーマ**: ライト/ダークテーマ切替、カスタムカラースキーム対応。

#### セッション管理
- **状態保存**: 最後のクエリ/フィルタ/カラム設定を `~/.alopex/tui-state.json` に保存し、次回起動時に復元。
- **複数タブ**: 複数のクエリ/テーブルを同時に開いてタブ切替で操作。
- **履歴**: 実行したクエリ/コマンドを履歴として保存し、再利用可能。

## 将来拡張のための前提
- プラガブルなトランスポート/認証（gRPC/HTTP、mTLS/OIDC など）を前提に抽象化。
- クラスタメタデータの取得 API が変化しても CLI 側はアダプタで吸収できる構造にする。
- `alopex-cli` 自身のバージョンとサーバー/クラスタの互換性マトリクスを持ち、警告を出せるようにする。

## 非対象（当面の非ゴール）
- GUI/ブラウザ UI。
- ワークフロー定義やジョブスケジューラ的機能（別ツールと連携想定）。
