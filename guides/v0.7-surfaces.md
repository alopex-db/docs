# v0.7 公開サーフェス

このページは、Alopex DB v0.7.0 で公開されたサーフェスと、v0.7.x
パッチ系列で維持される範囲を記録する。ここにない将来構想を、現行版の
CLI/API として扱ってはならない。

## Cluster-aware サーバー

v0.7 は **single-node compatible な cluster-aware foundation** である。
クラスタの状態・ノードID・membership lifecycle・SQL routing 診断を観測できるが、
SQLを別ノードへ送る実行、Raftによるメタデータ合意、分散トランザクションは
このリリースの実装範囲ではない。ノード跨ぎの実行は v0.8 以降の計画である。

### HTTP 管理 API

現行の管理エンドポイントは次のとおり。

| Method | Endpoint | 用途 |
|---|---|---|
| `GET` | `/api/admin/status` | サーバーとcluster status |
| `GET` | `/api/admin/health` | health status |
| `GET` | `/api/admin/metrics` | metrics |
| `POST` | `/api/admin/cluster/join` | ローカルノードのjoin lifecycle遷移 |
| `POST` | `/api/admin/cluster/leave` | ローカルノードのleave lifecycle遷移 |

`join` と `leave` は、v0.7では seed discovery やリモートノード追加を行うAPI
ではない。単一メンバーのローカル状態遷移として扱う。

### CLI

サーバー管理CLIは次のコマンドである。

```text
alopex server status
alopex server metrics
alopex server health
alopex server join
alopex server leave
```

`alopex-cluster init --nodes 3`、`join --seed`、`migrate --live`、
`sharding enable` は現行製品のコマンドではない。

### 設定

`cluster_aware` モードでは、ノードの識別情報を明示する。

```toml
[cluster]
mode = "cluster_aware"
node_id = "node-a"
cluster_id = "cluster-a"
advertised_endpoint = "127.0.0.1:8080"
role = "gateway"
membership_source = "chirps"
membership_source_available = true
```

設定ファイルの代わりに、例えば `ALOPEX__CLUSTER__MODE` のような
`ALOPEX__` 接頭辞と二重アンダースコア区切りの環境変数も利用できる。
既定値は `mode = "single_node"` である。

### Python の境界

組み込みPython APIの `Database.cluster_status()` は、組み込みエンジンから
観測できる単一ローカルノードのスナップショットを返す。
`routing_diagnostics()` も読み取り専用のローカル実行診断であり、リモート
クラスタへのjoinや分散実行を開始するAPIではない。

## DataFrame P3

string / datetime / list の名前空間と `explode` / `implode` は、Alopex DB
v0.7.0で出荷済みのDataFrame P3サーフェスである。これは
`alopex-dataframe v0.4.0` の将来機能ではない。

Rust APIでは `Expr::str()`、`Expr::dt()`、`Expr::list()` と、
`DataFrame::explode()`、`DataFrame::implode()`、`LazyFrame::explode()`、
`LazyFrame::implode()` が利用できる。Python APIでは `DataFrame.str()`、
`DataFrame.dt()`、`DataFrame.list()`、`DataFrame.explode()`、
`DataFrame.implode()` が対応する。

実装済みの名前空間関数は次の範囲である。

| Namespace | 実在する関数 |
|---|---|
| `str` | `to_lowercase`, `to_uppercase`, `contains`, `replace`, `strip_chars`, `split`, `len_chars`, `extract` |
| `dt` | `year`, `month`, `day`, `weekday`, `to_string`, `convert_time_zone` |
| `list` | `join`, `len`, `contains` |

## SQL parser と WASM

現行のSQL parserはNim実装をC ABI FFIで `alopex-sql` に統合する方針である。
WASM parserはv0.7の提供サーフェスではない。Read-Only SQLを含むWASM対応は
v1.0以降に再評価する計画であり、現行版の実装済み機能として案内しない。
